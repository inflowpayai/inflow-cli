# Security Policy

## Reporting a vulnerability

If you believe you've found a security issue in `inflow`, please **do not** open a public GitHub issue.

Email `security@inflowpay.ai` with:

- A description of the issue and its impact.
- Steps to reproduce (or a proof of concept).
- The affected package(s) and version(s).
- Any mitigating factors you're aware of.

We'll acknowledge receipt within two business days and aim to provide a remediation timeline within five.

## Scope

`@inflowpayai/inflow` (the published binary) and `@inflowpayai/inflow-core` (workspace-internal).
